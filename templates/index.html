Welcome to the Codon Compression web tool

Saturation mutagenesis is employed in protein engineering and genome-editing efforts to generate libraries that span amino acid design space. Our codon compression web tool allows users to compress degenerate codons in a user specified manner. The output provided will be a unique collection of codons for any desired amino acids.

This tool’s purpose is to eliminate undesired amino acids, stop codons and redundancy (or bias) from the generated diversity, as occurs in the commonly used NNK codon (that includes 32 codons for 20 amino acids and includes also stop codon).

The resulting compressed codons are optimized for the target organism codon usage, and custom genetic and usage tables can be uploaded for exotic organisms or when non-natural amino acids are in use.

This tool contains 2 different compression approaches:

DYNAMCC_0: This algorithm calculates the minimal set of degenerate codons for any desired set of amino acids, without redundancy. Note that the compressed codons may be ranked according to codon usage, or according to their usage rank, which may result in different results.

For example: to include all amino acids, without redundancy and without any stop codon, optimized for E. coli, the compressed codon list is: VHG, NRT, WTT and TGG

DYNAMCC_R: This algorithm helps in finding compressed codons with various levels of redundancy. Due to the large solution space, the algorithm compresses the “best List” according to usage that constitutes a single codon for each amino acid, and then adds more codons ordered by their specific usage, until complete redundancy is achieved.

For example: to include all amino acids, without any stop codon, optimized for E. coli, the compressed and fully redundant codon list is: NYA, NBG, VRA, NNY and VAG

Additionally, we have provided the capability to "explode" a set of compressed codons:

DYNAMCC_D: Here the user can define the Hamming distance between the wild type codon and the mutated ones. The aim of this algorithm is to further reduce library size, while still presenting relevant results. The rationale is that it has been shown by us and others that in many protein engineering studies the best performing mutant, (or the highest peak in the mutational landscape) requires more than a single base change in the codon. Alternatively, researchers who are interested in SNP research, such as in the case of cancer and other genetic diseases, may want to design their libraries to include a single base change only.

Codon Exploder: This tools allows you to visualize the mapping of compressed codons to the corresponding collection of true codons. The usage and ranking of the "exploded codons" will be determined based on the selected organism and displayed in the output table.
